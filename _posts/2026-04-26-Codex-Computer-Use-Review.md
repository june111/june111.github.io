# 实测 Codex Computer Use

听说codex的自动操作电脑很厉害，我试了一下，任务完成度非常震撼！但是一开始很艰难，又踩了个大坑...

我的电脑系统是 macOS 13，一调用 computer use 就崩了，弹出错误报告。

问 AI 排查不出问题，翻遍 Google，也没找到答案。

我反复把错误报告发给gpt，在反复检查授权，重装 codex，重装 computer use 5次后，终于知道了原因。

这个功能对系统有硬性要求，最低得是 macOS 15。

没办法，为了这口醋，只能升级系统了...

升级完后，终于可以用computer use。

我的任务很明确：导入素材，生成字幕，剪辑视频。

因为之前 Vibe Coding 踩过太多坑，这次我想让 AI 少走弯路，于是把每一个操作步骤都写得清清楚楚，心想一定要一次过。

任务开始后，相当惊艳，AI 像真人控制电脑一样，准确点击。

中途因为播放卡顿，屏幕跳出一个“优化性能”的弹窗。

AI 也检测到了，自己点了关闭提示。

这是预料之外的提示，我还以为要思考很久，谁知道一下子就知道要关闭。

最困难的任务是剪辑。

我让 AI 把视频最后 2 秒删掉，它在时间轴上反复定位、横跳，找了很久。

这一套流程跑下来，要半个多小时。

虽然产出结果完全符合预期，但也直接把我的 Token 额度烧干了，要等几个小时才能继续用。

这是我的 GPT 第一次用剪映，像一个有老花眼的年轻人。

明显的按钮，点得快准狠；但遇到精细的点位，就得盯着屏幕找很久。

虽然效率上还有很大进步空间，但看着鼠标自己移动，我感觉到了流水线工人被 AI 替代的必然。

这是属于我们这个时代的技术革命。
